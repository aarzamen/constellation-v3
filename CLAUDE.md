# CLAUDE.md — Agent Orientation for Constellation V3

## Project Overview

Constellation is a **self-hosted semantic memory server** for AI conversation history exports (currently Claude-only). It provides:
1. A local embedding + clustering pipeline (Python/numpy)
2. An MCP-compatible memory server for LLM clients (via `fastmcp` over stdio)
3. A REST API for search/retrieval
4. An interactive 3D visualization frontend (`3d-force-graph` + Three.js)

**Single developer project, MIT licensed, no CI/CD pipeline.**

## Architecture

```
launch.py
    ├── core/parser.py      → Parse Claude JSON export, chunk user messages
    ├── core/embedder.py    → sentence-transformers wrapper (all-MiniLM-L6-v2, 384d)
    ├── core/indexer.py     → KMeans clustering, PCA projection, graph generation
    ├── core/math_utils.py  → Pure numpy: kmeans++, silhouette, cosine sim, PCA
    ├── core/lexical.py     → BM25 inverted index (pure Python)
    ├── core/config.py      → YAML config management
    │
    ├── server/api.py       → SearchEngine class (hybrid RRF search, notes)
    ├── server/http_server.py → stdlib HTTPServer + REST endpoints
    └── server/mcp_server.py  → FastMCP stdio server (standalone subprocess)

frontend/
    ├── index.html          → Single page, loads CDN libs + local JS
    ├── css/constellation.css → Dark theme design system (~787 lines)
    └── js/
        ├── app.js          → Orchestrator, data loading, controls
        ├── graph.js        → 3d-force-graph + THREE.LOD rendering
        ├── search.js       → Client keyword + server hybrid search
        ├── inspector.js    → Conversation detail panel
        ├── timeline.js     → Brushable timeline (canvas 2D)
        └── starfield.js    → Background star renderer
```

**Total codebase: ~3,900 lines** (Python ~1,500 / JS ~1,040 / CSS ~790 / Tests ~320 / HTML ~180).

## Key Design Constraints

- **No external vector DBs** — all math is numpy-based (kmeans, PCA, cosine sim)
- **No frameworks** — no LangChain, LlamaIndex, Flask, FastAPI, React, etc.
- **No build step** — frontend is vanilla JS served from filesystem
- **MCP over stdout** — `mcp_server.py` uses stdio; ALL print statements MUST go to `sys.stderr` to avoid corrupting the JSON-RPC protocol
- **Chunk-level embeddings** — user messages are chunked (~150 words) and individually embedded; search operates on chunks then maps back to conversations

## Code Conventions

- **Python**: No type checker or linter configured. Uses standard library `http.server`. Imports are absolute from project root (`from core.config import ...`).
- **JavaScript**: Vanilla ES5/ES6 globals (no modules). Functions are file-scoped globals shared across `<script>` tags. No linter configured.
- **CSS**: Single stylesheet, CSS custom properties for theming. Dark mode only.
- **Logging**: All server-side `print()` statements in MCP-adjacent code MUST use `file=sys.stderr` to protect stdout for JSON-RPC.

## Known Issues and Gotchas

1. **`lexical.py:36` dead variable**: `np_zeros = [0] * self.doc_count` — the name `np_zeros` is assigned but is actually being assigned to `self.doc_lengths` via a chained assignment. Harmless but confusing; `np_zeros` is never referenced again.
2. **`inspector.js:41` dropped `query` parameter**: `renderInspector(node, conv, query)` is called from `showInspector` with 3 args, but the function signature is `renderInspector(node, conv)` — the `query` parameter is silently dropped, breaking query highlighting in the chat viewer.
3. **Palette arrays duplicated**: The DEFAULT_PALETTE and COLORBLIND_PALETTE arrays exist in both `core/indexer.py` (Python) and `frontend/js/graph.js` (JS) with no single source of truth.
4. **No CORS preflight handling**: `http_server.py` adds `Access-Control-Allow-Origin: *` but doesn't handle OPTIONS preflight requests — cross-origin POST to `/api/search` may fail from external clients.
5. **Config merge is shallow-ish**: `config.py:ensure_config()` does a one-level deep merge. Nested config beyond 2 levels won't merge properly.
6. **`parser.py:101` prints to stdout**: `parse_claude_export` uses bare `print()` (not stderr). Safe in the launch pipeline but would corrupt MCP if ever called from MCP context.
7. **HTTP server is synchronous**: `http.server.HTTPServer` is single-threaded; search requests block all other requests (including static file serving) while the embedding model runs.
8. **No request size limits**: `http_server.py:93` reads `Content-Length` bytes with no upper bound check.
9. **XSS in search results**: `search.js:86` injects `n.id` directly into an `onclick` attribute string. If a conversation ID contained a quote character, it could break out of the attribute.
